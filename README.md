# Зав. Склад

Веб-приложение для учёта прихода товаров цеха (ткань, фурнитура, расходники)
по 5 участкам: Закройный, Швейный, Утюг, Петля и пуговица, ОТК и упаковка.
Работает в реальном времени через Firebase, устанавливается на телефон как
приложение (PWA).

## Структура файлов

```
index.html          — разметка
style.css            — стили
app.js                — вся логика приложения
firebase-config.js     — сюда вставляешь свои ключи Firebase
manifest.json            — манифест PWA (установка на телефон)
sw.js                      — service worker (кэш для установки)
icons/icon-192.png          — иконка приложения 192×192
icons/icon-512.png          — иконка приложения 512×512
```

## Шаг 1. Настройка Firebase

1. Зайди на https://console.firebase.google.com и создай новый проект.
2. Слева выбери **Build → Firestore Database** → «Создать базу данных» →
   любой регион ближе к тебе → на первом этапе можно выбрать **тестовый режим**.
3. (Опционально, для фото) Слева выбери **Build → Storage** → «Начать».
4. Зайди в **Project settings** (шестерёнка) → вкладка **General** → внизу
   раздел «Your apps» → нажми на значок `</>` (Web) → зарегистрируй приложение
   (имя любое, Hosting не нужен) → скопируй объект `firebaseConfig`.
5. Открой файл `firebase-config.js` и вставь свои значения вместо
   `"ВСТАВЬ_СЮДА"`.

### Правила безопасности Firestore

Приложение работает без логина, поэтому доступ к базе открыт для всех, у кого
есть ссылка на приложение. Это осознанный компромисс ради простоты. Зайди в
**Firestore Database → Rules** и вставь:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
```

Если позже понадобится защита — можно будет добавить пароль или Firebase
Authentication отдельным этапом.

### Правила Storage (если подключал, для фото)

**Storage → Rules**:
```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /{allPaths=**} {
      allow read, write: if true;
    }
  }
}
```

### Индекс Firestore

При первом открытии каждого раздела и отчётов Firestore может попросить
создать составной индекс — в консоли браузера (F12) появится готовая ссылка,
просто перейди по ней и нажми «Создать». Это займёт пару минут, один раз.

## Шаг 2. Иконки

Добавь в папку `icons/` два файла (как делали раньше):
- `icon-192.png` — 192×192 px
- `icon-512.png` — 512×512 px

Без них PWA не установится корректно на телефон — сама разметка (manifest.json,
index.html) уже их ожидает.

## Шаг 3. Публикация на GitHub Pages

1. Создай новый репозиторий на GitHub, загрузи туда все файлы этой папки
   (включая `icons/` с готовыми иконками).
2. В репозитории: **Settings → Pages → Source** → выбери ветку `main` и
   папку `/root` → Save.
3. Через минуту GitHub выдаст ссылку вида
   `https://имя-пользователя.github.io/имя-репозитория/` — это и есть рабочая
   ссылка на приложение.

## Шаг 4. Установка на телефон

Открой ссылку на телефоне в браузере:
- **Android (Chrome)** — появится баннер «Добавить на главный экран», либо
  через меню (⋮) → «Установить приложение».
- **iPhone (Safari)** — кнопка «Поделиться» → «На экран «Домой»».

После установки иконка «Зав. Склад» появится на рабочем столе телефона и
будет открываться как обычное приложение.

## Как устроены данные

- **entries** — каждая запись прихода (партия). У ткани — свой набор полей
  (рулоны, цена за ярд, доллары), у остальных товаров — свой (количество,
  единица измерения, цена, сомы).
- **items** — справочник товаров по категориям, пополняется автоматически,
  когда вводишь новое название при добавлении прихода.
- **counters** — счётчики номеров партий, отдельно по каждой категории
  (ЗК-0001, ШВ-0001 и т.д.).
